# AI 개발 표준 v2.0

## 1. 개요

본 문서는 개발 조직에서 AI를 활용한 개발 생산성 향상 및 일관된 품질 유지를 위해 정의된 표준이다. AI는 단순 도구가 아닌 개발 조직의 일부(Agent)로 간주하며, 가상자산 및 결제 시스템의 특수성을 반영하여 엄격한 검증 및 보안 기준을 적용한다.

v2.0부터 **3-Agent 아키텍처**를 도입하여 설계, 실행, 검증의 책임을 명확히 분리한다:
- **Claude** — 설계 + 코드 생성 ("무엇을 만들지")
- **Codex** — 실행 + 수정 + 코드 리뷰 ("돌아가게 만듦")
- **Harness** — 테스트 + 검증 + 피드백 라우팅 ("맞는지 확인")

## 2. 기본 철학

### 2.1 역할 분리 원칙 (Mandatory)

> 코드 생성 ≠ 코드 검증

하나의 AI가 생성과 검증 역할을 동시에 수행하는 것을 엄격히 금지한다.

### 2.2 Human-in-the-loop

AI는 보조가 아닌 실행 주체이나, 최종 책임과 승인 권한은 인간 개발자에게 있다.

### 2.3 반복 개선 구조 (Feedback Loop)

실패 → 실패 사유 프롬프트 주입(Context Injection) → 재요청 → 개선

최대 3회까지 자동 재시도하며, 반복 실패 시 인간 개발자에게 이관한다.

### 2.4 테스트 기반 검증 (Test-Driven AI)

AI가 작성한 모든 비즈니스 로직은 반드시 단위 테스트(Unit Test) 코드를 동반해야 하며, 이를 통과해야만 리뷰 단계로 넘어갈 수 있다.

## 3. AI 조직 구조 (3-Agent 아키텍처)

```
[PM / 개발자]
      ↓
[Goal 정의]
      ↓
[Planner Agent]
      ↓
 ┌────────────┬─────────────┬─────────────┐
 ↓            ↓             ↓
Claude      Codex         Harness
(설계+생성) (실행+수정)    (테스트+검증)
      ↓            ↓             ↓
      └────────────┼─────────────┘
                   ↓
         [GitLab / Jira / Slack]
```

## 4. Agent 정의

### 4.1 Planner Agent

- **역할**: 요구사항 분석, Task 분해, Jira 티켓 자동 생성
- **출력 형식**: Task 1~N (API 설계, DB 구조, 서비스 로직, 통합 테스트 등)

### 4.2 Claude (Dev Agent)

- **역할**: API/로직 설계, 비즈니스 로직 코드 생성, 단위 테스트 케이스 설계
- **핵심**: "무엇을 만들지" 결정하는 **설계자**
- **규칙**: Linter 준수, 대규모 리팩토링 금지, 테스트 커버리지 80% 이상, 외부 라이브러리 추가 금지

### 4.3 Codex (Execution + Review Agent)

- **역할**: Claude의 코드를 빌드/실행하여 오류 수정, 코드 리뷰(검증) 수행
- **핵심**: "실제로 돌아가게" 만드는 **실행자**
- **규칙**: 설계 변경 금지, 실행 레벨 수정만 허용, 외부 라이브러리 추가 금지
- **리뷰**: 표준 리뷰 + [FINANCIAL] 태그 시 적대적 리뷰(adversarial-review)

### 4.4 Harness (Control Layer)

- **역할**: 3-Agent 오케스트레이션, CI 테스트 검증, 피드백 라우팅, 상태 관리
- **피드백 라우팅**: 실행/CI 실패 → Codex에게, 리뷰 실패 → Claude에게
- **상태**: TODO → DESIGN → CODEX_EXEC → CI_TEST → REVIEW → DONE → FAILED

## 5. 개발 프로세스 (3-Agent)

1. **Goal 정의** — PM/개발자가 목표 설정
2. **Task 분해** — Planner Agent가 Jira 서브 태스크 생성
3. **설계 + 코드 생성** — Claude가 API/로직 설계 및 코드 생성
4. **실행 + 수정** — Codex가 빌드/실행하고 오류 수정
5. **CI 검증** — Harness가 GitLab CI 파이프라인 실행 및 검증
6. **코드 리뷰** — Codex가 코드 리뷰 수행 ([FINANCIAL]은 적대적 리뷰)
7. **피드백 루프** — 리뷰 REJECT → Claude 재설계 / 실행 실패 → Codex 재수정 / 3회 실패 → FAILED
8. **배포** — Human 승인 후 Merge, Slack/Jira 알림

## 6. GitLab 규칙

- **브랜치**: `feature/AI-{task-id}`
- **PR 필수 조건**: CI 통과 + Review Agent 승인 + Human 승인 (금융 로직은 2인 이상)

## 7. 보안 정책

- 민감 정보의 Prompt 주입 및 로그 저장 절대 금지
- 금융 로직 격리: Human 2인 이상 크로스 체크 필수
- AI의 임의 외부 라이브러리 추가 금지

## 8. 금지 사항

- 하나의 AI 모델이 코드 생성과 검증을 동시에 수행
- AI 코드를 Human 최종 리뷰 없이 운영 환경에 직배포
- Human 승인 없는 대규모 구조 변경 및 외부 라이브러리 도입
